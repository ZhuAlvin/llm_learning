建议阅读顺序：

2. 再看模块 README（`notebooks/ModuleXX_*/README.md`）确认模块目标与最低完成标准

- 学习者：用报告识别本模块的高风险知识点

## 🎯 评估维度

每个 notebook 从以下 6 个维度进行评估（满分 100）：

1. **内容完整性** (20 分)
   - 是否覆盖所有必需的知识点
   - 内容的广度和深度

2. **理论深度** (20 分)
   - 数学推导是否严谨
   - 理论讲解是否清晰

3. **代码质量** (20 分)
   - 代码是否可运行
   - 注释是否详细
   - 是否遵循最佳实践

4. **实践练习** (15 分)
   - 微实践的数量和质量
   - 综合项目的完整性

5. **可视化** (10 分)
   - 图表的数量和质量
   - 是否帮助理解概念

6. **工程实践** (15 分)
   - 是否包含工程化技巧
   - 是否有生产级实践

**注**: Module 01-03 的"工程实践"维度不适用（0 分），因为它们是基础理论模块。
