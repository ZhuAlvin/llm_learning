  🔴 P0：必须修复

  1. 学习时间估计不一致（影响 M05/M07/M08）

  - M05 README 与 notebook 内标注有 1h 出入

  2. 评分体系缺乏依据（影响所有模块）

  - 所有 README 中的 notebook 评分（如 97/100、100/100）均无评分标准说明
  -
  建议：要么移除数值评分，要么在项目根目录补充评分维度定义（如覆盖度、实用性、代码质量等权重）

  3. 代码数值稳定性问题（影响 M02/M08）

  - M02 02_attention_mechanism.ipynb：使用 -1e9 代替 -float('inf') 做 attention masking
  - M08 01_rag_systems.ipynb：cosine 相似度未处理零向量、NDCG 分母可能为零
  - 建议：统一使用 -float('inf') 或 torch.finfo().min，补充零向量检查

  🟡 P1：重要优化

  4. README 与 LEARNING_GUIDE 职责重叠（影响所有模块）

  - 每个模块都有 README.md +
  LEARNING_GUIDE.md，两者在业务场景、学习目标、前置条件等内容上大量重复
  - 建议：在 LEARNING_GUIDE 开头添加交叉引用说明分工，或合并为一份

  5. 业务场景映射深度不均（影响 M01-M09）

  - 部分 notebook 的"电商客服智能助理"映射仅停留在一句话描述
  - 缺少真实对话数据示例和具体的业务指标阈值（困惑度多少算可用？准确率多少能上线？）
  - 建议：每个 notebook 补充至少一个真实客服对话示例 + 业务可用性阈值参考

  6. 异常处理普遍缺失（影响 M02/M06/M08/M09）

  - 教学代码几乎没有 try-except，文件读取、模型加载等操作无容错
  - M08 02_agent_systems_mcp.ipynb：使用裸 except: 捕获所有异常
  - 建议：在关键操作（文件 I/O、模型推理）处添加基本异常处理示范

  7. 术语记号跨 notebook 不统一（影响 M02/M04）

  - M02：Attention 在 02_attention 用 q,K,V，在 03_seq2seq 用 s_{t-1}, h_i
  - M04：README 用 "Masked Language Modeling"，notebook 有时用"掩码语言建模"，有时混用
  - 建议：统一为"中文术语（English Term, 缩写）"格式，首次出现时完整展开
