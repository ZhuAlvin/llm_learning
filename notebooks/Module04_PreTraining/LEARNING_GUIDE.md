# Module 4: 预训练 (Pretraining) - 学习手册

## 1. 本模块解决什么问题

你将理解“大模型先学通用规律，再做任务适配”的核心逻辑：

- 语言模型到底在学习什么分布？
- BERT (Bidirectional Encoder Representations from Transformers) 与 GPT (Generative Pre-trained Transformer) 该如何选？
- 预训练目标与业务效果之间是什么关系？

统一业务主线：`电商客服知识理解与回复生成`。

---

## 2. 学习前确认

- 完成 Module 3，能解释 Encoder/Decoder 差异
- 理解分类与生成任务基本评估指标
- 能运行最小文本训练与推理脚本

---

## 3. 统一学习动作

`业务问题 -> 任务定义 -> 最小实验 -> 核心公式 -> 能力边界 -> 选型建议`

---

## 4. 章节学习地图

| 章节 | 先回答的问题 | 生活化示例 | 最低完成标准 | 产出物 |
|---|---|---|---|---|
| `01_language_modeling.ipynb` | 模型如何估计下一个词或被遮蔽词？ | 预测客服用户常见追问表达 | 完成 CLM (Causal Language Modeling) / MLM (Masked Language Modeling) 小实验并对比困惑度 | 评估记录 |

| `03_gpt_architecture.ipynb` | 为什么 GPT 更擅长连续生成？ | 自动回复草稿与知识问答草案 | 完成一个 GPT 生成策略对比 | 生成质量对比 |

## 5. 学习路径

### 路径 A：任务选型优先（6-8 小时）

`01（核心） -> 02 或 03（按业务） -> 另一章重点阅读`

### 路径 B：标准完整（9-12 小时，推荐）

`01 -> 02 -> 03`

### 路径 C：策略深化（12+ 小时）

在路径 B 基础上增加：

- 比较 Prompt 与微调对效果/成本影响
- 增加数据规模变化实验
- 输出模型选型决策表

---

## 6. 学习曲线平缓策略

1. 先学“语言建模共同底层”，再分流到 BERT/GPT。
2. 不把架构之争抽象化，始终回到“理解任务 vs 生成任务”场景。
3. 每章最后都写一句“我会在什么需求下用它”。

---

## 7. 模块验收标准

- 能解释 CLM (Causal Language Modeling) 与 MLM (Masked Language Modeling) 区别。
- 能给出 BERT/GPT 在本项目中的选型依据。
- 能独立完成一个小型预训练或微调实验并复盘。
- 能说明至少 2 个预训练能力边界（时效性、事实一致性等）。

---

## 8. 常见卡点与应对

- 卡点：困惑度下降但回答质量不稳定。  
  应对：增加任务指标（准确率、人工可读性）联合评估。
- 卡点：BERT 与 GPT 不知怎么选。  
  应对：先判断任务输出类型：分类/抽取优先 BERT，连续生成优先 GPT。
- 卡点：实验结论不可复现。  
  应对：固定随机种子并记录数据切分。
