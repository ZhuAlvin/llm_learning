# Module 7: 部署与优化 (Deployment & Optimization) - 学习手册

> 📖 **本指南与 README 的关系**：README 提供模块概览和快速导航，本学习指南则提供详细的学习路径、深度解析和实践建议。建议先阅读 README 了解模块全貌，再按本指南深入学习。

## 1. 本模块解决什么问题

训练完成不等于业务可用，本模块聚焦上线阶段三问：

- 如何让模型响应更快、成本更低？
- 如何把模型变成稳定服务，而非单次脚本？
- 出现线上事故时如何快速止损并恢复？

统一业务主线：`电商客服在线服务在高并发场景下稳定运行`。

## 6. 学习曲线平缓策略

1. 先做“能跑起来”的最小服务，再谈大规模优化。
2. 先做关键指标可观测，再做复杂自动化。
3. 每次部署改动都配一个回滚预案。

## 7. 模块验收标准

- 能提交一份推理优化前后对比（延迟/吞吐/精度）。

- 能配置最小监控与告警链路。
- 能在演练中完成一次故障回滚。

---

## 8. 常见卡点与应对

- 卡点：单次请求快，但并发下崩溃。  
  应对：增加连接池、限流和动态批处理测试。
- 卡点：优化后精度掉得太多。  
  应对：给每种压缩策略设置精度下限阈值。
- 卡点：日志很多但定位慢。  
  应对：统一 trace_id，串联请求链路日志。
