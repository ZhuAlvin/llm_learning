# Module 1: 基础知识 (Foundations)

## 📚 模块概览

本模块是全课程的起点，目标不是“堆公式”，而是建立可迁移的底层直觉：为什么模型会学会、为什么会学偏、为什么会训练不稳。

为降低抽象感，本模块使用统一生活化主线：
`电商客服智能助理`。你会逐步看到数学、模型和训练技巧如何直接影响业务效果。

### 🎯 学习目标

- 掌握深度学习所需的数学基础（线性代数、微积分、概率论）
- 理解神经网络的基本原理和工作机制
- 熟悉PyTorch的核心功能和编程范式
- 能够从零实现简单的神经网络模型
- 掌握反向传播算法的原理和实现

### ✅ 完成本模块后的可交付产出

- 一个可运行的客服消息分类最小模型（含训练与评估）
- 一份客服模型训练诊断记录（过拟合、学习率、梯度稳定性）
- 一套从 NumPy 到 PyTorch 的客服分类实现对照笔记

### ⏱️ 预计学习时间

**总计**: 12-15 小时

### 📈 学习曲线设计（平缓爬坡）

- 第 1 段（1.1-1.2）：先理解”如何用向量表达客服消息”和”模型如何判断消息类别”
- 第 2 段（1.3-1.4）：再理解”客服分类模型如何学习”和”如何用 PyTorch 高效实现”
- 第 3 段（1.5）：最后整合为”可训练、可比较、可解释”的客服分类完整实验

### 🧭 每章建议阅读顺序

## ❓ 常见问题

### Q1: 数学基础不好可以学习深度学习吗？

**A**: 可以。本模块专门设计了数学基础回顾部分，从基本概念讲起。建议：
- 重点理解核心概念和直觉
- 多做微实践巩固理解
- 参考推荐的数学资源

### Q2: 为什么需要学习反向传播？

**A**: 反向传播是深度学习的核心算法，理解它有助于：
- 掌握模型训练的基本原理
- 排查训练中的问题
- 设计和改进模型架构
- 理解梯度消失/爆炸等问题

### Q3: NumPy和PyTorch有什么区别？

**A**: 主要区别：
- **NumPy**: 纯CPU计算，手动实现反向传播
- **PyTorch**: 支持GPU加速，自动微分，内置神经网络层
- 学习路径：先理解NumPy实现，再使用PyTorch

### Q4: 如何选择激活函数？

**A**: 经验法则：
- 隐藏层：ReLU及其变体（避免梯度消失）
- 输出层：分类用Softmax，回归用Linear
- 二分类：Sigmoid

### Q5: 反向传播的计算复杂度是多少？

**A**: 时间复杂度：O(n)，其中n是网络参数数量
- 前向传播：O(n)
- 反向传播：O(n)
- 与网络深度呈线性关系
