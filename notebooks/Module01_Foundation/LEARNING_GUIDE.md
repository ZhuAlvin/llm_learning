# Module 1: 基础知识 (Foundations) - 学习手册

> 📖 **本指南与 README 的关系**：README 提供模块概览和快速导航，本学习指南则提供详细的学习路径、深度解析和实践建议。建议先阅读 README 了解模块全貌，再按本指南深入学习。

## 1. 本模块解决什么问题

你会解决 3 个基础但高频的问题：

- 模型为什么能把文本变成可计算对象？
- 模型为什么会“学不会”或“学偏”？
- 从 NumPy 原型迁移到 PyTorch 后，工程效率为什么显著提升？

统一业务主线：`电商客服消息分类与回复建议`。

---

## 2. 学习前确认

开始前建议具备：

- Python 基础语法与函数封装
- NumPy 基础数组操作
- 高中数学到大学初等线性代数直觉

建议环境：Python 3.9+，可运行 `numpy/matplotlib/torch`。

---

## 3. 统一学习动作（每章都按这条链路）

`问题 -> 直觉 -> 最小实现 -> 公式 -> 评估 -> 取舍`

目标是先建立“能解释”的理解，再追求“能推导”的严谨。

## 5. 学习路径（按时间预算）

### 路径 A：最短闭环（6-8 小时）

`01 -> 02 -> 04 -> 03（关键节） -> 05（对比实验）`

适用：先要能做小项目，再回补理论细节。

### 路径 B：标准完整（12-15 小时，推荐）

`01 -> 02 -> 03 -> 04 -> 05`

适用：希望后续模块中少走弯路。

### 路径 C：理论加强（15+ 小时）

在路径 B 基础上增加：

- 手写更多梯度推导
- 增加 2 组超参数敏感性对比
- 输出失败案例与修复记录

## 6. 学习曲线如何保持平缓

建议按三段推进：

1. 第 1 段（1.1-1.2）：先把“表示+分类”跑起来，避免一开始陷入证明细节。
2. 第 2 段（1.3-1.4）：再把“为什么会收敛/不收敛”讲清，并迁移到工程框架。
3. 第 3 段（1.5）：最后做能力边界与业务成本的对照分析。

## 7. 模块验收标准（Definition of Done）

完成 Module 1 后，你至少应达到：

- 能从零解释一次前向与反向传播，不依赖黑盒术语。
- 能独立训练一个小型文本分类模型并做误差分析。
- 能说明 2 个常见训练失败原因及对应修复动作。
- 能给出“浅层/深层”在一个真实需求下的选型建议。

---

## 8. 常见卡点与应对

- 卡点：公式看懂但不会用。  
  应对：每看完一个公式，立刻写 10-20 行最小代码验证。
- 卡点：loss 下降但业务指标不提升。  
  应对：同时记录准确率、召回率、误判样本，避免只盯单一指标。
- 卡点：迁移到 PyTorch 后结构混乱。  
  应对：强制拆成 `dataset/model/train/eval` 四段结构。
